# Техническое задание
## Мини-проект: Консольное приложение "Крестики-нолики" с алгоритмом Минимакс
### Цель проекта: Создать консольное приложение, которое принимает текущее состояние игрового поля 3×3 и находит оптимальный ход для текущего игрока, используя алгоритм Mini-Max.

1. **Описание задачи**:
   - Программа принимает ввод - текущее состояние поля (3 X 3).
   - Определяет, кто сейчас должен сделать ход.
   - Находит оптимальный код для игрока с помощью алгоритма Mini-Max.
   - Выводит координаты клетки и оценку хода.
2. **Входные данные**:
   - Программа принимает:
     + Строку из 9 символов `X.O.X..O.`
     + Или 3 строки по 3 символа
     + Символы `X`, `O` и `.` (пустая клетка на поле)
3. **Логика**:
   - Определение игрока (того, кто должен сейчас сделать ход):
     + Если количество `X`и `O` равно, то ходит `X`
     + Если количество `X` > `O`, то ходит `O`
     + Количество `O` не может быть больше по условию (так как `X` начинают игру)
   - Принцип работы самого алгоритма Mini-Max:
     + В алгоритме имеются 2 игрока: `X - max; O - min`.
     + `Max` стремится максимизировать результат, а `Min`, в свою очередь, уменьшить.
     + Поэтому буду вводить систему кнута и пряника:
       * (+ 100 points) за победу `X` - максимальная пользая для первого игрока и вообще максимальная польза для `Max`
       * (- 100 points) за выигрыш `O` - максимальная польза для `Min`
       * (0 points) за ничью - никому не полезно и не выгодно.
4. **Выходные данные**: 
   - Координаты хода `(row, col)`.
   - Оценка хода (+ 100 points, - 100 points, 0 points).
   - Если игра уже завершена, то вывести Игра завершена. Победил `X` (или `O`, в зависимости от случая)
